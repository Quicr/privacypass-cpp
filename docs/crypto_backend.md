# Crypto Backend Selection

privacypass-cpp supports **OpenSSL 1.1+** (including 3.x) and **BoringSSL** as crypto backends, selected at compile time.

## Supported OpenSSL Versions

| Version | Status | Notes |
|---------|--------|-------|
| OpenSSL 1.1.1 | Supported | Uses legacy RSA APIs |
| OpenSSL 3.x | Supported (default) | Uses EVP APIs where available |
| BoringSSL | Supported | API differences isolated in compat layer |

## Quick Start

### OpenSSL (default — uses system OpenSSL)

```bash
cmake -B build
cmake --build build -j$(nproc)
./build/privacy_pass_tests
```

### OpenSSL with specific version

```bash
# Point at a specific OpenSSL installation
cmake -B build -DOPENSSL_ROOT_DIR=/path/to/openssl
cmake --build build -j$(nproc)

# Or via just
just test-openssl11 dir=/usr/local/opt/openssl@1.1
just test-openssl3 dir=/usr/local/opt/openssl@3
```

### BoringSSL

Build BoringSSL from source first:

```bash
git clone https://boringssl.googlesource.com/boringssl
cd boringssl && cmake -B build -GNinja -DCMAKE_BUILD_TYPE=Release
cmake --build build --target crypto ssl
cd ..
```

Then point privacypass-cpp at it:

```bash
cmake -B build-boringssl \
  -DPRIVACY_PASS_CRYPTO_BACKEND=boringssl \
  -DOPENSSL_ROOT_DIR=$(pwd)/boringssl \
  -DOPENSSL_INCLUDE_DIR=$(pwd)/boringssl/include \
  -DOPENSSL_CRYPTO_LIBRARY=$(pwd)/boringssl/build/libcrypto.a \
  -DOPENSSL_SSL_LIBRARY=$(pwd)/boringssl/build/libssl.a

cmake --build build-boringssl -j$(nproc)
./build-boringssl/privacy_pass_tests
```

## CMake Options

| Option | Values | Default | Description |
|--------|--------|---------|-------------|
| `PRIVACY_PASS_CRYPTO_BACKEND` | `auto`, `openssl`, `boringssl` | `auto` | Crypto backend selection |

In `auto` mode, the build checks for `openssl/is_boringssl.h` in the include path. If found, BoringSSL is selected; otherwise OpenSSL.

## Compile Definitions

The active backend is exposed via a compile definition on `privacy_pass_crypto`:

- `PRIVACY_PASS_WITH_OPENSSL` — OpenSSL 3.x backend
- `PRIVACY_PASS_WITH_BORINGSSL` — BoringSSL backend

## Architecture

```
src/crypto/
├── common.hpp     # RAII wrappers for OpenSSL/BoringSSL C types
├── compat.hpp     # Thin compatibility layer bridging API differences
├── init.cpp       # secure_clear + backend init/shutdown
├── hash.cpp       # SHA-256, SHA-384, HMAC, HKDF
├── random.cpp     # RAND_bytes
├── blind_rsa.cpp  # Blind RSA (RFC 9474)
└── voprf.cpp      # VOPRF P-384 (RFC 9497)
```

All API differences between OpenSSL and BoringSSL are isolated in `compat.hpp`. The source files are shared — no per-backend duplication. Backend selection is purely compile-time via `#ifdef` in the compat layer, with zero runtime overhead.

## Known Differences

- **RSA-PSS SPKI**: OpenSSL uses OID `1.2.840.113549.1.1.10` (rsaPSS) for RSA-PSS keys. BoringSSL uses plain RSA OID `1.2.840.113549.1.1.1`. SPKI bytes generated by one backend may not be parseable by the other.
- **Key generation**: Keys generated by either backend are functionally interoperable at the protocol level (blinded messages, signatures, tokens all work cross-backend).

## Testing Across Backends

Use `just` to test against all supported crypto variants:

```bash
just test-openssl          # System OpenSSL (whatever version is installed)
just test-openssl11        # OpenSSL 1.1 (default: /usr/local/opt/openssl@1.1)
just test-openssl3         # OpenSSL 3.x (default: /usr/local/opt/openssl@3)
just test-boringssl        # BoringSSL
just test-all-crypto       # All three in sequence
```

Override paths as needed:

```bash
just test-openssl11 dir=/opt/openssl-1.1
just test-openssl3 dir=/opt/openssl-3.2
just test-all-crypto openssl11_dir=/opt/openssl-1.1 openssl3_dir=/opt/openssl-3.2
```

You can also set the `OPENSSL_DIR` environment variable for ad-hoc use:

```bash
OPENSSL_DIR=/opt/openssl-1.1 just test
```

CI automatically tests both OpenSSL 1.1.1 and 3.x on Linux (built from source for 1.1).

## Benchmarking

Run benchmarks with backend labels:

```bash
./build/privacy_pass_benchmarks --benchmark_filter="BM_Provider_"
./build-boringssl/privacy_pass_benchmarks --benchmark_filter="BM_Provider_"
```
